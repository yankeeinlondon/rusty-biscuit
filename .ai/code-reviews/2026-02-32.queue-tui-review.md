# Queue TUI Design Suggestions

These notes focus on interaction design and terminal UX for the Queue TUI.

## Visual Hierarchy & Layout
- Add a true empty state in the main table (e.g., a centered line like "No tasks scheduled. Press N to add one.") so the UI has a clear first-action cue instead of an empty grid. (`queue/cli/src/tui/render.rs`)
- Make the table layout responsive to terminal width: collapse or shorten columns (e.g., hide "Status" or abbreviate "Where") when `area.width` is narrow to avoid truncation of the command field. (`queue/cli/src/tui/render.rs`)
- Provide a compact footer variant that truncates or wraps shortcuts based on available width; the current fixed list can overflow on 80x24 or smaller panes. (`queue/cli/src/tui/render.rs`)

## Color & Non-Color Fallbacks
- Avoid color-only meaning: the history list uses colored symbols (e.g., green check/red cross). Add ASCII fallbacks when `NO_COLOR` or limited color is detected (e.g., "[OK]", "[FAIL]") to preserve meaning. (`queue/cli/src/tui/history_modal.rs`)
- In the quit dialog, add brackets around keys ("[Y] Yes  [N] No") so the action is readable without color. (`queue/cli/src/tui/modal.rs`)

## Input & Focus Feedback (Ratatui patterns)
- Instead of inserting a `|` into the text value, use `frame.set_cursor_position` to render a real cursor for the active text field. This avoids altering the displayed content and keeps copy/paste clean. (`queue/cli/src/tui/input_modal.rs`)
- Keep placeholder text visible in a dim color while the field is active and empty, and place the cursor at the start. This improves clarity of expected input. (`queue/cli/src/tui/input_modal.rs`)
- For compact mode, consider shorter labels ("Cmd", "When", "Where") when width is tight; the fixed `COMPACT_LABEL_WIDTH` wastes horizontal space in narrow terminals. (`queue/cli/src/tui/input_modal.rs`)

## History Modal Ergonomics
- In compact mode, when a filter is active but the filter section is hidden, surface the active filter in the help line (e.g., "Filter: git | Esc: Close") so users can see what is narrowing the list. (`queue/cli/src/tui/history_modal.rs`)
- Add a count summary (e.g., "12 of 58") near the filter or title to improve scanning and confirm filter impact. (`queue/cli/src/tui/history_modal.rs`)

## Terminology & Labels
- Replace the "bg" target label with "background" when space allows, or use a more explicit short label ("bg" is ambiguous to new users). (`queue/cli/src/tui/render.rs`)
- Consider adding a short status legend in the footer or title when symbols are used (e.g., "✓ done, ✗ failed") to reduce cognitive load. (`queue/cli/src/tui/history_modal.rs`)
